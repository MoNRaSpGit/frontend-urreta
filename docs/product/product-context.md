# Urreta - Contexto inicial del producto

Fecha de actualizacion: 2026-05-16

## Contexto

`frontend-urreta` nace como base nueva de frontend para un proximo modulo o producto de `SaasPro`.

## Estado de definicion actual

Hoy el producto todavia no tiene contrato formal de backend ni definicion funcional cerrada.

Eso significa:

- el frontend se usa para explorar direccion de producto
- la base visual ya se esta ordenando
- las primeras interacciones sirven para validar lectura y operacion

## Lectura actual del modulo

Lo que hoy existe sugiere una direccion de trabajo centrada en:

- clientes
- pedidos
- registro
- productos

Pero esa lectura todavia no se considera definitiva.

## Regla de este corte

Antes de bajar integracion formal con backend conviene definir:

- objetivo exacto del modulo
- entidades base reales
- flujo principal que el cliente va a usar
- prioridad entre demo local y contrato real

## Meta de la documentacion en esta etapa

Como el modulo recien arranca, la documentacion busca:

- dejar una base limpia
- no sobreinventar arquitectura de producto
- registrar con claridad lo poco que ya esta definido
